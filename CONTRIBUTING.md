# Contributing

1. Fork the repository and create your branch from `main`.
2. Follow the existing architectural patterns defined in `docs/architecture.md`.
3. If changing BLE payloads, update `docs/protocol.md`.
4. If adding new hardware or pins, update `docs/wiring.md`.
5. Keep UI changes within `lvgl_ui.h` or isolate them to specific header files.
6. Submit a Pull Request.

Ensure all markdown files contain the standard Navigation header.
